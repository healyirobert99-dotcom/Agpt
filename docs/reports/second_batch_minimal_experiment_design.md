## 4. 为什么不建议直接再跑大规模旧空间

以下为基于已有证据的合理推断，并非绝对结论，不排除未来在改变条件后产生不同结果：

1. **第一批 94 个公式在 selection 阶段整体收益/风险指标全部为负值。** total_return（均值 -0.406）、sharpe（均值 -0.908）、sortino（均值 -1.409）、max_drawdown（均值 -0.542）。不一致性极低：94 个公式中没有一个出现正收益。

2. **fast screen 的 IC 指标已显示旧空间的系统性方向偏差。** 在 selection 阶段，94 个公式的 `rank_ic_mean` 仅有 12 个为正值，82 个为负值。虽通过 fast screen 的低阈值（min_abs_rank_ic_mean 仅为 0.001），但 IC 方向整体偏负。

3. **如果完全不改变表达空间（特征 5 个 × 算子 10 个 × 公式长度限制 8 token），继续机械扩大搜索量可能只是增加计算成本。** 现有 94 个代表性公式已覆盖该空间下相关性去重后的主要模式变体。该空间在 selection 时段（2023-2024）表现整体失效，同样的空间扩产更多公式大概率产生类似结果。

4. **现有证据不支持判定旧空间"永久无效"。** 因子研究存在统计不确定性，不同市场阶段的公式表现可能不同。当前结论仅针对 selection 时段（2023-2024）。

## 5. 第二批最小实验设计

以下设计遵循"小实验优先"原则，避免一次性大规模搜索。

目的是：在**可控的增量成本**下验证"旧空间是否还有未被挖掘的信号"，或在新方向下快速验证可行性。

### 5.1 实验 0：旧空间验证（带中等数量新公式）

如果用户选择沿用当前特征/算子空间：

| 项目 | 建议规模 |
|---|---|
| 生成新公式数 | 200-500（而非 10000） |
| 相关性去重后目标 | 50-100 个 |
| 快速筛选 → 回测 → 评级 | 全链路走通 |
| 预期耗时估算 | （待用户批准后由系统估算） |

**实验 0 的两个可能结果：**

- **结果 A：selection 阶段仍然 0 个通过。** → 直接确认旧空间在当前 selection 时段的表达能力接近极限，不需要再继续扩大数量。
- **结果 B：出现少量正收益公式。** → 放大该子方向生成更多变体进行第二轮筛选。

**规模选项（需用户审批）：**

| 选项 | 说明 | 建议 |
|---|---|---|
| 200 个公式 | 最小验证量，6-8 小时完成完整链路 | ✅ 建议优先考虑 |
| 500 个公式 | 中等规模，比 200 个覆盖更广 | 可作为备选 |
| 1000+ 个公式 | 接近小规模批量，不建议第一步就做 | ❌ 不推荐 |

### 5.2 实验 1：仅扩展有限算子（不扩展基础特征）

如果用户批准扩展算子，最小的可行方向：

在现有 10 个算子的基础上，**仅增加 1-2 个新增算子**（具体算子见第 6 节），使用同一批基础特征生成 200-500 个新公式。

**原理：** 在不引入新数据依赖的前提下，新增算子可以提供新的变化模式，但保留原有空间的可控性。由于算子新增完全基于本地现有数据（原特征经过新算子变换），不需要引入外部数据。

### 5.3 实验 2：新增本地可派生的基础特征（不引入外部数据）

如果用户批准扩展基础特征，最小的可行方向：

仅增加 1-2 个**使用本地已有日线数据即可直接计算**的新特征（例如衍生自 MAX/MIN、简单均线差、振幅等）。这些特征不需要引入新的数据源或外部数据。

使用更新后的特征空间（5+1 或 5+2 个特征），在原有算子下生成 200-500 个新公式。

### 5.4 实验之间的执行顺序

建议按以下顺序执行（每个实验的结果决定是否继续下一个）：

```
实验 0：旧空间 + 200 新公式
   ├── 如果有正收益选股通过 → 放大该方向
   └── 如果 0 个通过 → 进入实验 1

实验 1：旧特征 + 新增算子 + 200 新公式
   ├── 如果有正收益选股通过 → 放大该方向
   └── 如果 0 个通过 → 进入实验 2

实验 2：新增基础特征 + 旧算子 + 200 新公式
   ├── 如果有正收益选股通过 → 放大该方向
   └── 如果 0 个通过 → 暂停，提交阶段性报告
```

### 5.5 早期停止条件

每个实验的结果在**第一阶段**（fast screen 完成后）如果没有正 IC 方向的公式通过，可以直接报告并停止，无需继续 Phase 2 回测。

早期停止的具体条件：

| 阶段 | 停止条件 | 建议 |
|---|---|---|
| fast screen 后 | 通过公式中没有任何正 `rank_ic_mean` 的条目 | 建议停止该实验，不进入 Phase 2 |
| 相关性去重后 | 去重后候选公式少于 3 个 | 建议停止 |
| Phase 2 回测后 | 全部 `total_return <= 0` | 该实验确认无效，进入下一个实验 |

### 5.6 无合格因子时的结果输出

所有实验的结果允许为 0。当产生 0 个合格因子时，系统如实输出 0 个。

严禁为了得到 A/B 结果而：

- 降低 fast screen 的 `min_abs_rank_ic_mean`
- 放宽 `total_return > 0` 的评级门槛
- 回填不存在的指标
- 改变评级规则使某些公式"恰好通过"

### 6.4 关于外部数据的说明

以下方向**明确不作为当前最小实验的候选**：

- 外部数据源（估值、财务、分析师预期等）— 需要引入新数据，当前建议聚焦本地数据

## 7. 不建议纳入最小实验的内容

以下内容不适合在第二批最小实验中立即引入，理由围绕**降低无效工作量和保持研究口径一致性**：

| 不建议的内容 | 理由 |
|---|---|
| **复杂模型（ML/DL）** | 引入黑箱模型会增加理解和解释成本，偏离本项目"公式表达"的核心原则。应先验证扩展后的搜索空间是否有信号，再考虑是否需要复杂模型。 |
| **高频数据 / 分钟级数据** | 需要额外数据获取和预处理流水线，且验证周期长。最小实验应保持与第一批一致的数据频率。 |
| **大规模参数搜索（网格/随机/贝叶斯）** | 第一批万级搜索的经验表明，大规模但不改变表达空间的搜索可能只是增加计算成本。应在空间扩展有初步结果后再考虑参数优化。 |
| **过多新特征一次性加入（> 3 个）** | 一次性加入过多特征会混淆变量，无法判断哪个方向有效。最小实验每次只改变 1-2 个维度。 |
| **过多新算子一次性加入（> 2 个）** | 同上。每次增量变动应保持可追溯性。 |
| **为了"提高命中率"而放宽筛选或评级标准** | 宽松标准获得的"假阳性"只会增加噪音，不增加因子的真实价值。"没有合格因子"是可接受的研究结果。 |
| **券商接入 / 自动交易 / 买卖建议** | 项目目标明确排除自动交易。本实验仅为因子搜索研究。 |
| **访问 forward data** | forward data 应保留作为真正的样本外验证，不能用于因子生成或筛选。 |
| **降低 min_coverage / min_abs_rank_ic_mean** | 当前 fast screen 阈值已经很低（min_abs_rank_ic_mean = 0.001），进一步降低会引入大量无意义公式。现有数据已证明 fast screen 不是瓶颈—94 个全部通过但后续回测全部失败。 |
